<objective>
Fix the single wrong column reference in `list_admin_reports` that prevents ward names from ever appearing in the admin triage queue.

Purpose: Plan 04 added `LEFT JOIN wards ON wards.id = reports.ward_id` but selected `wards.name AS ward_name`. The wards table (defined in `004_ward_boundaries.sql` line 10) has no column called `name` — the column is `ward_name`. PostgreSQL rejects the query at runtime with "column wards.name does not exist", making ward_name null or error for every report. The unit test at line 855 also embeds the wrong string. This is a single-word surgical fix that closes WARD-01.
Output: One line changed in production SQL, one line changed in unit test SQL string, cargo test passes.
</objective>
