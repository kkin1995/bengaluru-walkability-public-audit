<objective>
Implement org-scoped report visibility so that an admin assigned to an organization sees only reports in wards belonging to that org's subtree, while admins with no org assignment see all reports.

Purpose: ROADMAP.md Success Criterion #3 requires "An admin user can be assigned to an organization, and that assignment controls which reports they see in the triage queue." The org assignment infrastructure is fully in place (FK column, assign endpoint, UI) but the scoping effect on report visibility is missing. JwtClaims does not carry org_id, so scoping must be implemented by fetching the admin's org_id from the DB on each request using claims.sub, then applying a recursive CTE WHERE filter when org_id is Some.

Output: list_admin_reports and count_admin_reports accept an `org_id: Option<Uuid>` parameter. When Some, a recursive CTE walks the organizations tree from that org downward and restricts rows to reports whose ward_id is linked to the subtree. The handler fetches the calling admin's org_id once per request before calling both queries.
</objective>

--   )
