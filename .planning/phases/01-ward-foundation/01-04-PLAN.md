<objective>
Close two UAT-blocking gaps in the admin reports API: (1) the JSON shape mismatch between backend and frontend, and (2) the missing ward_name in the report list query.

Purpose: The admin reports list and map pages both crash or show "No reports found" because the backend returns `{ items, page, limit, count }` while the frontend `AdminReportListResponse` interface expects `{ data, pagination: { page, limit, total_count, total_pages } }`. Fixing the backend shape resolves both pages. The ward JOIN is a secondary gap: even after the shape fix, ward_name would always be null because the SELECT never touches the wards table.

Output: A backend that returns the correct JSON shape with accurate pagination counts, a ward_name field in every report row, and a null guard in the map page as belt-and-suspenders.
</objective>
